# Configuration de Terra

## Qu'est-ce que Terra ?
Terra est un générateur de terrain open-source tiers, disponible sur [GitHub](https://github.com/PolyhedralDev/Terra). PNX intègre Terra comme solution de génération de terrain pour le serveur, vous permettant d'utiliser Terra dans PNX pour créer des niveaux épiques.

## Démarrage avec Terra
Pour appliquer Terra à votre monde, vous devez suivre ces deux étapes :
Dans `server.properties`, réglez :
```properties
use-terra=on
```
Créez un dossier dans le répertoire `worlds`, par exemple, `world/`, et créez le fichier de configuration suivant `config.json`.
```json
{
  "format": "leveldb",
  "generators": {
    "0": {
      "name": "terra",
      "seed": 0,
      "dimensionData": {
        "dimensionName": "minecraft:overworld",
        "dimensionId": 0,
        "minHeight": -64,
        "maxHeight": 319,
        "height": 384,
        "chunkSectionCount": 24
      },
      "preset": {
        "pack": "overworld"
      }
    }
  }
}
```
Le `preset/pack` représente le nom du paquet de cette configuration Terra, que vous pouvez trouver dans le champ `id` du fichier `pack.yml` à l'intérieur du pack de configuration. Les majuscules et les minuscules sont autorisées.
Ensuite, redémarrez le serveur pour expérimenter le générateur de terrain Terra.

## Utilisation des packs de configuration Terra Terrain
Terra vous permet de découvrir de nouveaux terrains en utilisant différents packs de configuration. Placez le pack de configuration dans le dossier `./terra/packs` et redémarrez le serveur pour terminer l'installation.
Vous pouvez vérifier les packs de configuration actuellement disponibles sur ce site : [Community Packs](https://terra.polydev.org/config/community-packs.html)
Ensuite, nous prendrons le pack de configuration [ReimagEND](https://github.com/justaureus/ReimagEND) comme exemple, avec le fichier de pack de configuration nommé "ReimagEND.zip".
Nous voulons appliquer ce pack de configuration dans la dimension de fin, alors créez un fichier `config.json` :
```json
{
  "format": "leveldb",
  "generators": {
    "2": {
      "name": "terra",
      "seed": 0,
      "dimensionData": {
        "dimensionName": "minecraft:end",
        "dimensionId": 0,
        "minHeight": 0,
        "maxHeight": 255,
        "height": 256,
        "chunkSectionCount": 16
      },
      "preset": {
        "pack": "REIMAGEND"
      }
    }
  }
}
```
Par exemple, dans ce cas, le fichier de pack de configuration est nommé ReimagEND.zip, nous pouvons vérifier le `pack.yml`, et définir le `preset/pack` sur `REIMAGEND` pour sélectionner ce pack de configuration.
Pour différents types de monde, les limites y varient comme suit :
- `minecraft:overworld`: -64 à 319 (chunkSectionCount: 24)
- `minecraft:nether`: 0 à 127 (chunkSectionCount: 8)
- `minecraft:end`: 0 à 255 (chunkSectionCount: 16)

Pour certains packs de configuration, il est possible que la limite de hauteur du monde soit inférieure à ce que le pack de configuration nécessite (par exemple, le pack Tartarus Nether). Il doit être appliqué à l'overworld et non au nether, car le nether est limité à un maximum de 128.

## Pas encore effectif
- Si votre monde utilisait précédemment un générateur différent de Terra, le paramètre ne prendra pas effet et vous devrez commencer avec un nouveau niveau.
- Vérifiez bien si vous avez correctement configuré Terra ; sinon, le paramètre ne prendra pas effet.

## Utilisation excessive de la mémoire
Si vous souhaitez réduire l'utilisation de la mémoire, vous pouvez essayer de modifier `./terra/config/yml` :

```yaml
cache:
  structure: 0 # Default: 32
  sampler: 0 # Default: 128
  biome-provider: 0 # Default: 32
```
